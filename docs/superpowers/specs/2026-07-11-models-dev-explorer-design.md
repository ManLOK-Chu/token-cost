# models.dev 模型探索与单文件拆分设计

日期：2026-07-11

## 1. 背景与目标

当前 `improve-models-dev-modal` 分支在 `index.html` 中实现了 models.dev 目录弹窗，已经能够按需加载模型、搜索并将价格填入计算器，但存在以下局限：

- 模型结果以卡片行展示，不利于横向比较多种价格。
- 只展示输入和输出价格，缺少 Context、缓存读取与缓存写入价格。
- 搜索仅使用简单包含匹配，没有厂商或缓存能力筛选。
- 动态价格的显示精度与主价格表不一致。
- 缺失价格和合法 `0` 没有被完整区分。
- 选择一行会立即应用，用户不能先比较再确认。
- HTML、CSS、计算、图表、存储和目录浏览逻辑全部集中在 `index.html`。

本次目标是提升桌面端模型查找与价格比较效率，并同步拆分现有单文件实现。成功标准如下：

1. 使用桌面价格比较表展示模型、厂商、Context 和四类价格。
2. 支持智能相关度搜索、厂商多选、缓存支持筛选和表头排序。
3. 用户先选择行，再通过固定底部操作栏确认使用模型。
4. 正确区分缺失价格与合法 `0`，避免图表中的错误价格回退。
5. 将 `index.html` 拆分为职责清晰、无循环依赖的原生 ES modules。
6. 保持零构建、无第三方运行时依赖，并确保现有计算器行为不回退。

本功能明确不提供移动端适配。现有主计算器的响应式行为保持不变。

## 2. 范围

### 2.1 本次包含

- 拆分内联 CSS 与 JavaScript。
- models.dev 目录数据的标准化。
- 搜索、智能排序、厂商多选和缓存支持筛选。
- 桌面价格比较表和可排序表头。
- Context、输入、输出、缓存读取、缓存写入列。
- 两步模型选择流程。
- 加载、失败重试、空结果和截断提示。
- 弹窗焦点管理、键盘交互和背景滚动锁定。
- Chrome DevTools MCP 驱动的真实浏览器验收。

### 2.2 本次不包含

- 移动端探索弹窗布局。
- 虚拟滚动或无限滚动。
- 将动态 models.dev 模型跨刷新持久化。
- 保存自定义预设到文件。
- 引入 React、Vue、npm 构建链或第三方数据表库。
- 重写现有成本公式或 SVG 图表算法。

## 3. 技术架构

项目继续使用原生 HTML、CSS 和 JavaScript，通过静态 HTTP Server 或 GitHub Pages 运行。

```text
index.html
styles.css
model-presets.json
js/
├── app.js
├── dom.js
├── pricing.js
├── storage.js
├── presets.js
├── chart.js
└── models-dev.js
```

### 3.1 `index.html`

- 只保留语义化页面结构。
- 使用 `<link rel="stylesheet" href="styles.css">` 加载样式。
- 使用 `<script type="module" src="js/app.js">` 加载应用入口。
- 不保留内联样式和业务脚本。

### 3.2 `styles.css`

- 机械迁移现有全部样式。
- 增加筛选工具栏、价格表、选中行、sticky 表头和底部确认栏样式。
- 删除只服务于旧卡片列表的样式。
- 不为探索弹窗增加移动端 media query。
- 探索弹窗设置保证七列表格可读的最小宽度；窄视口允许横向滚动。

### 3.3 `js/app.js`

- 作为唯一应用协调者。
- 控制初始化顺序。
- 注册价格、token、预设和主题相关事件。
- 连接 `models-dev.js` 的确认回调与 `presets.js`。
- 统一触发摘要、图表、模型选择器和价格参考表更新。

### 3.4 `js/dom.js`

- 提供 `$()` 等 DOM 查询辅助函数。
- 提供通用 HTML 转义和格式化函数。
- 不持有业务状态。

### 3.5 `js/pricing.js`

- 读取和标准化计算输入。
- 包含 `calculateCostAtRate()`、`buildSeries()` 等纯计算函数。
- 提供金额、token 和百分比格式化。
- 不依赖 DOM，便于通过浏览器 module import 单独验证。

### 3.6 `js/storage.js`

- 封装 `localStorage` 读取与写入。
- 管理主题、输入值、当前静态预设和图表可见模型。
- 不持久化动态 models.dev 模型。
- 恢复时忽略无法在静态预设中解析的动态 ID。

### 3.7 `js/presets.js`

- 加载 `model-presets.json`。
- 管理静态预设和本次页面生命周期中的动态预设。
- 渲染价格预设选择器、图表模型选择器和价格参考表。
- 对外提供查找、添加和枚举预设的接口。

### 3.8 `js/chart.js`

- 管理 SVG 坐标轴、曲线、当前命中率标记和 tooltip。
- 接收规范化计算数据和预设，不直接读取探索弹窗状态。
- 模型价格覆盖使用 `??` 或显式缺失检查，确保合法 `0` 不会回退到当前表单价格。

### 3.9 `js/models-dev.js`

- 加载和缓存 models.dev 目录。
- 标准化远程数据。
- 管理搜索、筛选、排序和临时选择状态。
- 渲染价格比较表。
- 通过 `onSelect(model)` 回调提交最终选择，不直接操作图表或主计算器。

模块通过显式导出函数和回调通信。`app.js` 是唯一协调者，其他模块不形成循环依赖。

## 4. 探索弹窗界面

### 4.1 顶部结构

弹窗依次包含：

1. 标题、来源说明和关闭按钮。
2. 搜索框。
3. 筛选工具栏。
4. 结果状态栏。
5. 可滚动价格比较表。
6. 固定底部确认栏。

### 4.2 搜索

搜索范围包括：

- 模型友好名称。
- models.dev 原始模型 ID。
- family。
- 厂商 ID 和显示名称。

搜索不区分大小写。输入按空白切分为多个词，所有词都必须在模型的组合搜索文本中匹配，即 AND 语义。搜索输入采用短延迟 debounce，避免每次按键立即重绘大量行。

### 4.3 筛选工具栏

#### 厂商筛选

- 使用多选下拉面板。
- 选项根据当前目录中实际存在的厂商生成。
- 未选择任何厂商表示不过滤。
- 工具栏显示已选厂商数量。

#### 缓存支持筛选

提供三个互斥选项：

- 全部。
- 支持缓存。
- 不支持缓存。

`cache_read` 或 `cache_write` 任一字段存在有效数值时，即视为支持缓存。字段缺失与数值 `0` 必须区分：`0` 是有效价格，因此表示该项价格已提供。

#### 清除筛选

- 清除搜索词、厂商选择和缓存支持筛选。
- 恢复智能排序。
- 清除临时选中行。

### 4.4 状态栏

显示：

- `匹配 N / 总计 M 个模型`。
- 价格单位 `USD / 1M tokens`。
- 数据来源 models.dev。
- 如果匹配数超过渲染上限，显示“当前显示前 300 个”等明确提示。

## 5. 价格比较表

列顺序固定为：

```text
模型 | 厂商 | Context | 输入 | 输出 | 缓存读取 | 缓存写入
```

### 5.1 单元格规则

- 模型列展示友好名称，并以次要样式展示原始模型 ID。
- 厂商列展示规范化显示名称。
- Context 使用 `K`、`M` 等紧凑格式。
- 价格保留原始有效精度，不使用强制 `toFixed()` 截断。
- 缺失值显示 `—`。
- 合法 `0` 显示 `$0`。
- 表格主体在弹窗内部滚动，表头保持可见。

### 5.2 表头排序

以下列支持点击排序：

- Context。
- 输入价格。
- 输出价格。
- 缓存读取价格。
- 缓存写入价格。

第一次点击按升序排序，再次点击同一列切换为降序。表头使用方向箭头和 `aria-sort` 表示状态。搜索词或筛选条件改变后保留当前手动排序方式。

无论升序还是降序，`null` 或缺失值始终位于末尾。合法 `0` 按普通数值参与排序。

## 6. 智能相关度排序

未手动选择排序列时采用智能相关度。存在搜索词时，优先级为：

1. 模型名称精确匹配。
2. 模型名称前缀匹配。
3. 模型名称或原始 ID 包含全部搜索词。
4. 厂商、family 等辅助字段匹配。
5. 价格字段更完整。
6. 支持缓存。
7. 厂商名称和模型名称的稳定字母顺序。
8. 原始目录索引作为最终稳定 tie-breaker。

没有搜索词时，优先展示价格字段完整且支持缓存的模型，再按厂商和模型名称稳定排序。

## 7. 两步模型选择

- 单击行只更新临时选中状态，不修改主计算器。
- 同一时间最多选择一行。
- 再次单击已选行仍保持选中，避免误取消。
- 选中行具有清晰背景、边框和 `aria-selected="true"`。
- 底部确认栏展示所选模型名称和厂商。
- 未选中模型时，“使用此模型”按钮禁用，并显示“请先选择一个模型”。
- 只有输入与输出价格均存在时才允许确认；否则按钮禁用并解释缺失字段。
- 点击“使用此模型”后：
  1. 将模型注册为本次会话的动态预设。
  2. 缺失缓存价格在计算器中填入 `0`，但预设说明明确标记为“目录未提供”，不宣称免费。
  3. 更新预设选择器、四个价格输入、摘要、图表、模型选择器和价格参考表。
  4. 关闭弹窗。
- 关闭弹窗时清空搜索、筛选、排序和临时选择。
- 重新打开时不默认选择上次模型。
- 已确认的动态预设保留到页面刷新为止。

## 8. 标准化数据模型

models.dev 条目转换为：

```js
{
  id,
  sourceId,
  name,
  providerId,
  providerName,
  family,
  contextWindow,
  priceInput,
  priceOutput,
  priceCacheRead,
  priceCacheWrite,
  supportsCache,
  searchText,
  originalIndex
}
```

规则：

- 缺失数值使用 `null`。
- 原始数值为 `0` 时保留 `0`。
- `supportsCache` 由缓存读取或写入字段是否为有效数值决定。
- 输入或输出价格缺失的模型仍出现在表格中，但不能确认使用。
- 应用内部 ID 由 provider ID 与原始模型 ID 生成，原始 ID 单独保留用于展示与搜索。

## 9. 状态与数据流

探索弹窗内部状态：

```js
{
  catalog,
  query,
  selectedProviders,
  cacheFilter,
  sortKey,
  sortDirection,
  selectedModelId,
  loading,
  error
}
```

数据流：

```text
打开弹窗
  → 按需 fetch models.dev
  → 标准化目录
  → 搜索 / 筛选 / 排序
  → 单击行形成临时选择
  → 点击“使用此模型”
  → models-dev.js 调用 onSelect(model)
  → presets.js 注册动态预设
  → app.js 应用价格并统一刷新
  → 关闭弹窗并重置探索状态
```

目录数据在当前页面生命周期内只加载和标准化一次。失败的请求不会被缓存，以允许重试。

## 10. 加载与错误处理

弹窗支持四种明确状态：

### 加载中

- 显示从 models.dev 拉取目录的状态。
- 暂时禁用筛选器和表格操作。

### 加载成功

- 显示结果总数、来源和价格单位。
- 自动将焦点放入搜索框。

### 空结果

- 保留当前搜索和筛选条件。
- 显示无匹配提示。
- 提供清除筛选操作。

### 加载失败

- 显示简洁错误摘要。
- 提供“重新加载”按钮。
- 重试前清除失败 Promise。
- 不影响静态预设和主计算器继续工作。

请求必须检查 HTTP 状态和响应的基本目录结构。界面提醒 models.dev 为数据来源，实际价格以供应商账单或官方价格页为准。

## 11. 可访问性与键盘操作

- 弹窗使用 `role="dialog"`、`aria-modal="true"` 和标题关联。
- 打开后焦点进入搜索框。
- 关闭后焦点返回“浏览 models.dev 模型”按钮。
- `Escape` 关闭弹窗。
- 打开时锁定背景页面滚动。
- `Tab` 和 `Shift+Tab` 被限制在弹窗可交互元素内。
- 使用原生 `<table>`、`<thead>`、`<th>` 和 `<tbody>` 语义。
- 可排序表头维护正确的 `aria-sort`。
- 行可聚焦，`Enter` 或 `Space` 选中。
- 行维护 `aria-selected`。
- 底部确认按钮支持标准键盘操作。

## 12. 性能

- 目录只下载并标准化一次。
- 搜索使用短延迟 debounce。
- 搜索、筛选和排序均在内存中完成。
- 首版最多渲染 300 行，并在状态栏明确提示截断。
- 不引入虚拟滚动。
- 排序使用稳定 tie-breaker，避免结果顺序跳动。

## 13. 桌面端约束

- 探索弹窗按桌面端设计，保证七列表格的信息密度和可读性。
- 不新增探索弹窗的移动端 media query、抽屉或卡片降级。
- 在小于最小宽度的视口中允许横向滚动，不压缩列内容。
- Chrome DevTools 验收使用 `1440×900` 和 `1920×1080` 两种桌面视口。
- 主计算器已有响应式样式保持不变。

## 14. 测试与验收

测试以 Chrome DevTools MCP 驱动真实浏览器为主，纯函数检查为辅。项目不引入 npm 测试框架。

### 14.1 运行环境

通过本地静态 HTTP Server 启动项目，再由 Chrome DevTools MCP 打开页面。不得使用 `file://`，以确保 ES modules、JSON 和远程请求行为与部署环境一致。

- SVG 图表成功绘制，不存在 `NaN` 或无效坐标。
- 主题切换和刷新持久化正常。

## 15. 实施与回归策略

实施分两个阶段：

1. 机械拆分现有 HTML、CSS 和 JavaScript，先确保页面行为与拆分前一致。
2. 在稳定模块边界上增强 models.dev 探索表格。

不在同一步骤中重写成本公式或图表算法。每个阶段完成后都通过真实浏览器检查 Console、Network 和关键计算结果，降低拆分与功能增强互相掩盖缺陷的风险。
